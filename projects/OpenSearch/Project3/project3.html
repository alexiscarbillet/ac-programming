Centralizing Kubernetes Logs Across GKE, AKS, and EKS Using OpenSearch

In modern cloud infrastructures, logs are produced everywhere: applications, containers, system services, firewalls, and managed cloud services. Centralizing these logs is critical for threat detection, compliance, debugging, and observability.

Below is a unified architecture showing how GKE, AKS, and EKS clusters forward their logs into a centralized OpenSearch SIEM platform.

Log Sources

You need sources that generate logs, such as:

  • Linux syslogs
  • Windows Event Logs
  • NGINX/Apache logs
  • Firewall logs
  • IDS/IPS logs (e.g., Snort, Suricata)

In Kubernetes clusters, logs mainly come from:

  • Container logs (stdout/stderr)
  • Kubelet logs
  • Control-plane logs (depending on cloud provider)
  • Network and audit logs
Log Shipping

To centralize logs from virtual machines or Kubernetes clusters, the most common and compatible tool is Filebeat, provided by Elastic and fully compatible with OpenSearch.

Install Filebeat on Linux:

sudo apt install filebeat
Shipping logs from Kubernetes clusters (GKS, AKS, EKS)

Deploy Filebeat as a DaemonSet on each cluster:

kubectl apply -f https://raw.githubusercontent.com/elastic/beats/8.6/deploy/kubernetes/filebeat-kubernetes.yaml

Configure Filebeat to output to OpenSearch:


output.opensearch:
  hosts: ["https://opensearch.yourdomain.com:9200"]
  username: "admin"
  password: "yourpassword"
  ssl.certificate_authorities: ["/etc/pki/root/ca.pem"]
Data Ingestion and Enrichment

Logs are ingested and enriched using Logstash. 


input {
  beats { port => 5044 }
  file {
    path => "/var/log/suricata/eve.json"
    codec => json
    type => "suricata"
  }
  file {
    path => "/var/log/ufw.log"
    start_position => "beginning"
    sincedb_path => "/dev/null"
    type => "firewall"
  }
}

filter {
  if "syslog" in [tags] {
    grok { match => { "message" => "%{SYSLOGLINE}" } }
    date { match => ["timestamp", "MMM  d HH:mm:ss", "MMM dd HH:mm:ss"] }
  }

  if "nginx" in [tags] or "apache" in [tags] {
    grok { match => { "message" => "%{COMBINEDAPACHELOG}" } }
    date { match => ["timestamp", "dd/MMM/yyyy:HH:mm:ss Z"] }
    geoip { source => "clientip" }
  }

  if [type] == "suricata" and [event_type] == "alert" {
    mutate { add_tag => ["suricata_alert"] }
  }

  if "winlogbeat" in [agent][type] or "windows" in [tags] {
    mutate { add_tag => ["windows_event"] }
  }

  if [type] == "firewall" {
    grok { match => { "message" => "%{SYSLOGTIMESTAMP:timestamp} %{IP:src_ip} .*DPT=%{NUMBER:dest_port}" } }
    date { match => ["timestamp", "MMM dd HH:mm:ss"] }
  }
}

output {
  if "syslog" in [tags] {
    opensearch { hosts => ["http://localhost:9200"] index => "syslog-%{+YYYY.MM.dd}" }
  } else if "nginx" in [tags] or "apache" in [tags] {
    opensearch { hosts => ["http://localhost:9200"] index => "web-logs-%{+YYYY.MM.dd}" }
  } else if "suricata_alert" in [tags] {
    opensearch { hosts => ["http://localhost:9200"] index => "suricata-%{+YYYY.MM.dd}" }
  } else if "windows_event" in [tags] {
    opensearch { hosts => ["http://localhost:9200"] index => "windows-logs-%{+YYYY.MM.dd}" }
  } else if [type] == "firewall" {
    opensearch { hosts => ["http://localhost:9200"] index => "firewall-logs-%{+YYYY.MM.dd}" }
  } else {
    opensearch { hosts => ["http://localhost:9200"] index => "misc-logs-%{+YYYY.MM.dd}" }
  }
}
Configuring Filebeat Tags

filebeat.inputs:
  - type: log
    paths: ["/var/log/syslog"]
    tags: ["syslog"]

  - type: log
    paths: ["/var/log/nginx/access.log"]
    tags: ["nginx"]

  - type: log
    paths: ["/var/log/apache2/access.log"]
    tags: ["apache"]
Index in OpenSearch

Create index templates for fields such as:

  • timestamp
  • source_ip / destination_ip
  • log_level
  • event_type
  • kubernetes metadata
  • cloud provider metadata
Dashboards

Use OpenSearch Dashboards to visualize:

  • GeoIP attack maps
  • Failed SSH login attempts
  • Top malicious IP addresses
  • Kubernetes pod activity
  • Suricata alerts
